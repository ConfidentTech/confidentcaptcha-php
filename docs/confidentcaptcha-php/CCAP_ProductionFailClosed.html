CCAP_ProductionFailClosed

Confident CAPTCHA policy for production, failures leave page closed

Occasionally, the call to create a Confident CAPTCHA will fail, due to upstream issues (failed API credentials, maintenance downtime, etc.). For a page like an account signup form, it is better that the page remains closed to all users than to let some automated accounts be created.

When there is an upstream issue, this policy will display a message that CAPTCHA creation failed, and to try again later.
